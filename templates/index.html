    How It Works

    01

    Detect

    25+ regex patterns catch SSNs, emails, phones, IDs. spaCy NER adds PERSON / GPE / ORG hits. Runs entirely on-premise.

    —
    02

    Encrypt

    Each PII value is AES-256-GCM encrypted with a unique nonce and stored in a local SQLite vault. The original text is never written to disk.

    —
    03

    Tokenise

    The document gets <<SV-TOKEN>> placeholders. It looks complete — just contains no real PII. Safe to share across teams.

    —
    04

    Authorise

    A doctor clicking <<SV-TOKEN-0041>> decrypts only the fields their role permits. Auditors see different fields. Nothing else leaks.

    06

    What Happens if the Vault is Stolen?

    🔐

    AES-256-GCM Encryption

    Every PII value is encrypted with a 256-bit key. Brute-force time: 2²⁵⁶ combinations — longer than the age of the universe.

    🏢

    On-Premise Only

    Vault, keys, and documents never leave your server. Air-gap compatible. ShadowVault's own servers see zero data.

    🗝

    Per-Role Key Isolation

    Each role has a separate derived key. Compromising one role's key exposes only that role's permitted fields.

    📋

    Full Audit Trail

    Every token decryption is logged: who, which role, which token, when. Immutable access log in the vault database.
